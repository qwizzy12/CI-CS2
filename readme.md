CI/CD & Observability Стек для Игровой Инфраструктуры (CS2)
Репозиторий содержит декларативное описание инфраструктуры игрового сервера и автоматизированный GitOps-пайплайн для непрерывной доставки (CD) конфигураций и игровых плагинов.

Технологический стек
ОС: Linux (Ubuntu Server)
Контейнеризация: Docker, Docker Compose
CI/CD: GitHub Actions, Rsync, SSH-action
Observability: Prometheus, Node Exporter, Grafana
БД: MySQL 8.0

Архитектура и Автоматизация

1. Инфраструктура как код (IaC)
Вся сопутствующая инфраструктура упакована в `docker-compose.yml`
MySQL 8.0 Используется для персистентного хранения игровой статистики и логов (настроены Docker Volumes для сохранности данных при перезапуске).
Prometheus: Сборщик метрик производительности системы и контейнеров.
Grafana: Визуализация метрик на кастомных дашбордах.

2.CI/CD Пайплайн (GitHub Actions)
Реализован автоматический деплой игровых модов и плагинов для фреймворка **CounterStrikeSharp** без ручного вмешательства:
При пуше в ветку `main` запускается workflow, использующий `rsync` для синхронизации локальной папки `plugins_and_configs` с целевой директорией на удаленном сервере Ubuntu.
Использование `rsync` с флагами оптимизации минимизирует сетевой трафик (передается только дельта изменений) и автоматически очищает удаленные плагины на сервере через `--delete`.
После успешной синхронизации через SSH-демона выполняется перезапуск системной службы `systemctl restart cs2.service` для горячей перезагрузки игрового сервера и применения модов.
